# Проект CI/CD-pipeline  

## Цель  
Целью проекта является разработка CI/CD-pipeline для стратегии развёртывания ML-модели Canary (канареечная модель). 

## Краткое описание стратегии  
В данном проекте стратегия Canary обеспечивает поэтапный перевод трафика со старой модели (stable) на новую (canary) и откат (при необходимости). Перераспределение трафика выполняется конфигами nginx. В проекте предусмотрено ручное переключение между конфигами. Решение по переключению может быть принято на основании A/B теста.  

## Краткая инструкция  
**1 Выполнение пайплайна CI/CD, в соответствии с кодом файла deploy.yml, происходит автоматически при выполнении в репозиторий [https://github.com/Roman-Golubev/HW7_CICD] push новых скриптов, моделей (pkl) и т.д.**  
**2 После успешного завершения развёртывания на виртуальной машине к ней возможно выполнять запросы на предсказание, например, с помощью команды `curl -X POST ...` - см. приложенный ноутбук.**  
**3 В проекте предусмотрены конфиги в папке nginx: nginx.10.conf, nginx.50.conf, nginx.100.conf, nginx.rollback.conf. По умолчанию в проекте используется nginx.10.conf - 10 % трафика направляется на новую модель.**  
**4 Для перераспределения трафика на виртуальной машине необходимо выполнять команды для соответствующих конфигов, например: `cp nginx/nginx.50.conf nginx/nginx.10.conf` -> docker restart nginx_canary.**
